# 🐾 ВетЗабота — Ветеринарная клиника

[![PHP](https://img.shields.io/badge/PHP-8.0+-777BB4?style=flat&logo=php&logoColor=white)](https://www.php.net/)
[![MySQL](https://img.shields.io/badge/MySQL-8.0-4479A1?style=flat&logo=mysql&logoColor=white)](https://www.mysql.com/)

**ВетЗабота** — современное веб-приложение для ветеринарной клиники с полным циклом работы: от регистрации пользователя до управления записями через админ-панель. Реализовано с фокусом на безопасность и удобство использования.

## 📦 Установка

### Требования
- XAMPP 8.0+ (или отдельно Apache + PHP + MySQL)
- Браузер Chrome/Firefox/Edge последних версий

### Шаг 1 — Скопировать файлы
```bash
cd C:\xampp\htdocs
git clone https://github.com/your-username/vetclinic.git
# или просто распакуйте архив в C:\xampp\htdocs\vetclinic
```

### Шаг 2 — Запустить XAMPP
1. Откройте **XAMPP Control Panel**
2. Запустите **Apache** и **MySQL**

### Шаг 3 — Создать базу данных
1. Откройте **phpMyAdmin**: `http://localhost/phpmyadmin`
2. Создайте базу: `vetclinic` (кодировка `utf8mb4_unicode_ci`)
3. Выберите базу → вкладка **SQL**
4. Выполните файлы по порядку:
   - `database.sql` — основная структура
   - `patch.sql` — поля brute-force защиты
   - `patch2.sql` — таблица логов
   - `patch3.sql` — расписание врачей
   - `patch4.sql` — индексы для оптимизации

### Шаг 4 — Проверить конфигурацию
Откройте `config/db.php` и убедитесь что настройки подключения соответствуют вашим:
```php
$host     = 'localhost';
$user     = 'root';
$password = '';
$database = 'vetclinic';
```

### Шаг 5 — Открыть сайт
```
http://localhost/vetclinic/
```

## 🛡 Безопасность

Проект реализует многоуровневую защиту:

### Защита от SQL-инъекций
Все запросы к БД используют **prepared statements** с `bind_param()`:
```php
$stmt = $db->prepare("SELECT * FROM users WHERE email = ? LIMIT 1");
$stmt->bind_param('s', $email);
```

### Защита от XSS
- Все выводимые данные экранируются функцией `e()` (обёртка над `htmlspecialchars()`)
- Content Security Policy с nonce для inline-скриптов

### Защита от CSRF
- Уникальный токен в каждой форме
- Проверка через `hash_equals()` для защиты от timing-атак

### Защита от Brute-force
- **Уровень 1**: блокировка аккаунта после 5 попыток на 15 минут
- **Уровень 2**: rate limiting по IP (20 попыток / 15 минут)

### Безопасное хранение паролей
- Алгоритм **bcrypt** через `password_hash()`
- Встроенная соль, адаптивная сложность

### Безопасное восстановление пароля
- Криптографически стойкий токен (`random_bytes(32)`)
- В БД хранится SHA-256 хеш токена
- Срок действия: 15 минут
- Одноразовое использование
